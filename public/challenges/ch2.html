📖 What is SQL Injection?

SQL Injection happens when user input is embedded directly into a SQL query without sanitization. The attacker can break out of the intended query using a single quote ' and inject their own SQL commands.

⚔ The query being run is:
SELECT name, open_time FROM gates WHERE name = '[YOUR INPUT]'

Try Main Gate first. Then try ' OR '1'='1 to dump all rows. Then use the UNION button to extract the hidden flag column.

  2. Try a single quote ' — observe the SQL error that leaks the query.
  3. Try ' OR '1'='1 — dumps all rows because WHERE is always true.

💡 The Fix

Parameterized queries
// ❌ VULNERABLE — string concatenation
const query = `SELECT * FROM gates WHERE name = '${name}'`;

// ✅ SECURE — parameterized query
const query = 'SELECT * FROM gates WHERE name = ?';
db.prepare(query).all(name);
// The DB treats name as pure data, never as SQL code
