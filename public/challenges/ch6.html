📖 What is Stored XSS?

Stored Cross-Site Scripting (XSS) happens when malicious JavaScript is saved to the database and later rendered unsanitized in another user's browser. Unlike reflected XSS, stored XSS persists — every visitor who loads the page executes the attacker's script automatically.

⚔ Post a message containing: <script>alert(document.cookie)</script>

Then click Load Board. The script executes in your browser showing your session cookie. In a real attack this cookie would be sent silently to the attacker's server. The flag appears once the board detects your payload executed.

💡 The Fix

Escape output
// ❌ VULNERABLE — renders raw HTML from database
div.innerHTML = userMessage;

// ✅ SECURE — treats content as plain text, not HTML
div.textContent = userMessage;

// ✅ ALSO SECURE — sanitize before rendering
const clean = DOMPurify.sanitize(userMessage);
div.innerHTML = clean;
