📖 What is Mass Assignment?

Mass Assignment occurs when an API blindly binds all fields from a request body to a database model — including sensitive fields like role, isAdmin, or balance that should only ever be set server-side. The attacker simply adds an extra field to the request body that was never meant to be user-controlled.

⚔ The POST /api/profile endpoint accepts username, role, and userId from the request body.

Try sending role: admin and watch what happens. The server will update your role to whatever you send — including admin.

🗺 Step-by-Step

  1. Set the User ID to 4 — that is your peasant account.
  2. Change the Role field from peasant to admin.
  3. Click Update Profile and observe the response.
  4. The server returns the flag when it detects you set your role to admin.

💡 The Fix

Allowlist only safe fields
// ❌ VULNERABLE — accepts everything including role
const { username, role } = req.body;
db.prepare('UPDATE users SET username=?, role=? WHERE id=?')
  .run(username, role, userId);

// ✅ SECURE — only allow safe fields, ignore the rest
const { username } = req.body; // role is never accepted from user
db.prepare('UPDATE users SET username=? WHERE id=?')
  .run(username, req.session.user.id);
