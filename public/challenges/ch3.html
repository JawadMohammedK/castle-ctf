📖 What is Broken Access Control?

Broken Access Control is the #1 vulnerability in OWASP Top 10 2021. It occurs when access restrictions are enforced using data that the client controls rather than the server. Here the server trusts a custom HTTP header X-Role that you send — meaning you can claim any role you want.

⚔ The server checks: if (req.headers['x-role'] !== 'admin') → deny

You control your request headers. Try adding X-Role: admin to access Room 1 (Throne Room). Try X-Role: mage to access Room 3 (Mage Tower).

🗺 Step-by-Step

  1. Try accessing Room 1 with role set to peasant — you will be denied.
  2. Change the role to admin and try Room 1 again.
  3. Change the role to mage and access Room 3.
  4. Both rooms contain flags inside their content field.

💡 The Fix

Server-side role from session
// ❌ VULNERABLE — trusts client-supplied header
const role = req.headers['x-role'];
if (role !== 'admin') return res.status(403).json({ error: 'Denied' });

// ✅ SECURE — role comes from server-side session only
const role = req.session.user.role;
if (role !== 'admin') return res.status(403).json({ error: 'Denied' });
