📖 What is JWT Tampering?

JWT (JSON Web Token) is a signed token containing a payload like your username and role. If the secret key is weak, attackers can brute-force it and re-sign the token with modified contents. Some servers also accept the alg: none algorithm which requires no signature at all.

💡 The Fix

Strong secret + strict algorithm
// ❌ VULNERABLE — weak secret, accepts none algorithm
jwt.verify(token, 'weakkey123', { algorithms: ['HS256', 'none'] });

// ✅ SECURE — strong random secret, only HS256 allowed
const SECRET = require('crypto').randomBytes(64).toString('hex');
jwt.verify(token, SECRET, { algorithms: ['HS256'] });
// Never use 'none' — it means no signature required at all
