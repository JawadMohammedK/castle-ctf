📖 What is Path Traversal?

Path Traversal (also called Directory Traversal) lets attackers use ../ sequences to navigate outside the intended directory and read arbitrary files — config files, source code, private keys, and more. The ../ means "go up one folder" in the file system.

⚔ The endpoint is /api/letter?file=welcome.txt

Step 1: Try king_secret.txt — it is in the same letters folder and contains the flag.
Step 2: Try ../server.js to read the actual server source code.
Step 3: Try ../database.js to see the database setup including all passwords.

💡 The Fix

Resolve and verify the path stays inside allowed directory
// ❌ VULNERABLE — joins path directly with user input
const filePath = path.join(__dirname, 'public/letters', req.query.file);
fs.readFileSync(filePath);

// ✅ SECURE — resolve full path then verify it stays inside allowed dir
const safeBase = path.resolve(__dirname, 'public/letters');
const filePath = path.resolve(safeBase, req.query.file);

if (!filePath.startsWith(safeBase)) {
  return res.status(403).json({ error: 'Access denied' });
}
