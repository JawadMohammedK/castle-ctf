"The castle's archivist stores a personal scroll for every inhabitant — their deepest secrets, their hidden allegiances. Each scroll is retrieved by a simple number: the inhabitant's ID. You have been given access to your own scroll as a peasant (id=4). But the archivist never checks who is asking — only what number they provide..."

📖 What is IDOR?

Insecure Direct Object Reference (IDOR) happens when an application uses a user-supplied value (like an ID in the URL) to directly access data, without checking whether the logged-in user is actually authorized to access that object.

⚔ Your mission: your scroll is at ?id=4. Change the ID to 1, 2, and 3 to access other users' scrolls. Each one contains a FLAG.

🗺 Step-by-Step

  1. Fetch your own scroll at ?id=4 — see what a normal response looks like.
  2. Change the ID to 1, 2, and 3.
  3. Each scroll contains a FLAG — copy it and submit it below.
  4. Notice how the server never asks who you are, only what ID you want.

💡 The Fix

Secure version
// ❌ VULNERABLE — trusts user input directly
app.get('/api/scroll', (req, res) => {
  const id = req.query.id;  // user controls this!
  const user = db.get('SELECT * FROM users WHERE id = ?', id);
  res.json(user);
});

// ✅ SECURE — always use the session user id
app.get('/api/scroll', (req, res) => {
  const id = req.session.user.id; // server controls this
  const user = db.get('SELECT * FROM users WHERE id = ?', id);
  res.json(user);
});
