The Scroll Vault

IDOR Insecure Direct Object Reference 100 pts each Novice
"The castle's archivist stores a personal scroll for every inhabitant — their deepest secrets, their hidden allegiances. Each scroll is retrieved by a simple number: the inhabitant's ID. You have been granted access to the castle as a peasant. The archivist will hand you your scroll. But what about the scrolls of others? The archivist is old, and trusting, and never questions who is asking — only what number they provide."

📖 Vulnerability Brief

Insecure Direct Object Reference (IDOR) occurs when an application exposes internal object identifiers — like database IDs — directly to the user, without verifying whether that user is authorized to access the referenced object. Your goal is to access data that does not belong to you.

⚡ The Scroll Reader

You are logged into the castle as a peasant. The endpoint below fetches a scroll by ID. Your task: find scrolls that do not belong to you and capture the flags within them.

GET /api/scroll?id=
USER ID

🔓 Solution & The Fix

How it works + the secure version
// The vulnerability — no ownership check
app.get('/api/scroll', (req, res) => {
  const id = req.query.id;  // user controls this directly
  const user = db.get('SELECT * FROM users WHERE id = ?', id);
  res.json(user); // returns ANY user's data
});

// The fix — always use the session
app.get('/api/scroll', (req, res) => {
  const id = req.session.user.id; // server controls this
  const user = db.get('SELECT * FROM users WHERE id = ?', id);
  res.json(user); // only ever returns YOUR data
});
