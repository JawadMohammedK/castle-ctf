📖 What is Brute Force / Missing Rate Limiting?

Brute Force attacks systematically try all possible values until the correct one is found. Without rate limiting or account lockout, a 4-digit PIN (0000 to 9999) can be cracked in seconds. Every login form, PIN pad, and OTP field needs protection against this.

💡 The Fix

Rate limiting + lockout
// ❌ VULNERABLE — no rate limit, no lockout, instant response
app.post('/api/vault', (req, res) => {
  const vault = db.get('SELECT * FROM vault WHERE pin = ?', [req.body.pin]);
  res.json(vault ? { treasure: vault.treasure } : { error: 'Wrong pin' });
});

// ✅ SECURE — rate limit + lockout after 5 attempts
const rateLimit = require('express-rate-limit');
const vaultLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 5,                    // max 5 attempts
  message: { error: 'Too many attempts. Vault locked for 15 minutes.' }
});
app.post('/api/vault', vaultLimiter, handler);
